Что это за система

WELLSIGHT/WS2000 — это система инженерного учета по скважине, где все ежедневные операции по буровым растворам ведутся через Daily Field Reports, а затем агрегируются в Post Well Audit. Ядро системы: баланс объемов + потерь/приростов + затрат + свойств раствора + гидравлики.

3) Ключевые workflow (от ежедневки до закрытия)

Workflow A: Запуск новой скважины

  • Создать Well Data, задать систему единиц, профиль, MD/TVD
  • Заполнить Well Casing и Well Sections
  • Создать базовые Fluid Sets и Product List
  • Проверить обязательные поля (*) перед первым Daily Report

Workflow B: Суточный отчет инженера БР

  • Создать Daily Report (Draft)
  • Ввести Wellbore + Bit run + Inventory + Costing
  • Ввести Volumes и Losses & Gains
  • Ввести Fluid Properties и Hydraulics (ECD)
  • Заполнить Time Breakdown + Issues
  • Запустить Finish Report (валидация) и закрыть день

Workflow C: Обязательная сверка балансов

Net Volume Change (Volumes) должен совпадать с Documented Net Volume Change (Losses & Gains).

При расхождении отчет помечается ошибкой и не должен уходить в completed/approved.

Workflow D: Post Well Audit

  • Сформировать интервалы между обсадками
  • Построить recap по затратам, объемам и типам раствора
  • Добавить инженерные выводы и рекомендации
  • Закрыть аудит (status = closed)

5) Критичные правила и валидации

R1: Все обязательные поля (*) заполнены до статуса completed.
R2: Нельзя менять titration method после начала серии анализов.
R3: Net change из Volumes = Documented net change из Losses & Gains.
R4: Нельзя отправлять на сервер отчеты Draft.
R5: Post Well Audit должен быть closed перед финальной передачей.
R6: Для компонентов в Product List обязателен specific gravity (SG).
R7: Tight Hole ограничивает доступ к данным скважины.

Green checks

Баланс объема сошелся, ECD в пределах, costs согласованы.

Warnings

Неполные поля, отклонения свойств раствора от target range.

Blocking errors

Несходимость объемов, отрицательные остатки, missing mandatory.

6) Отчеты, обмен и локализация

Отчеты и экспорт

  • Daily report / Summary / Recap Costs / Recap Volumes
  • Форматы: PDF, XLSX, DOCX (или HTML-предпросмотр)
  • Данные выгружаются в server inbox, затем публикуются

Локализация

  • 2 уровня: регион ОС + язык отчетов в приложении
  • Поддержка: RU, EN, ES, DE, PT (из руководства)
  • Интерфейс может оставаться EN, язык отчета настраивается отдельно
