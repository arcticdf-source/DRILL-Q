# 14. Формула расчета Success Fee

## 1) Базовая модель

Success Fee привязан к подтвержденному экономическому эффекту (Verified Value).

Формула:

Success Fee = alpha * max(0, Verified Value - Threshold)

Где:

- alpha: доля Исполнителя в эффекте (например, 10-25%)
- Verified Value: подтвержденный экономический эффект за период
- Threshold: минимальный порог эффекта, после которого включается success fee

## 2) Расчет Verified Value

Verified Value = Value_NPT + Value_ESP - Additional OPEX

Где:

- Value_NPT: экономия от снижения NPT
- Value_ESP: экономия от предотвращенных отказов ЭЦН и простоев
- Additional OPEX: дополнительные затраты на реализацию рекомендаций

## 3) Пример расчета

Входные данные за месяц:

- Value_NPT = 180,000 USD
- Value_ESP = 140,000 USD
- Additional OPEX = 40,000 USD
- Threshold = 100,000 USD
- alpha = 15%

Промежуточный расчет:

- Verified Value = 180,000 + 140,000 - 40,000 = 280,000 USD
- Eligible Value = 280,000 - 100,000 = 180,000 USD
- Success Fee = 0.15 * 180,000 = 27,000 USD

## 4) Опциональные модификаторы

### 4.1) Градация по достижению KPI

- Если KPI-порог достигнут на 100-110%: alpha = 15%
- Если KPI-порог достигнут на 111-130%: alpha = 18%
- Если KPI-порог достигнут > 130%: alpha = 22%

### 4.2) Защитный cap

- Success Fee cap: не более [X]% от monthly subscription

### 4.3) Защитный floor

- Минимальный успех-платеж при достижении KPI: [фиксированная сумма]

## 5) Юридические и учетные оговорки

- Методика расчета Verified Value утверждается до старта пилота
- Все исходные данные трассируемы и доступны обеим сторонам
- Спорные расчеты передаются в joint finance review

## 6) Рекомендуемые коммерческие параметры для первых пилотов

- alpha: 12-18%
- Threshold: 50,000-150,000 USD в месяц (по масштабу актива)
- Settlement period: ежемесячно
- Final reconciliation: по итогам 90 дней
