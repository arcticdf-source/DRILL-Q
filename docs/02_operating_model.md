# 02. Операционная модель (полностью цифровая)

## 1) Принципы

- Zero-paper: акты, наряды, маршруты, HSE-чеклисты только в цифровом виде
- API-first: все системы интегрируются через API и event bus
- Data-first: любое решение обосновывается данными
- AI-assisted by default: каждый инженерный процесс поддержан AI-помощником

## 2) Цифровой контур операций

- Command Center 24/7 (удаленные инженеры + AI-алерты)
- Mobile Field App (офлайн/онлайн): задания, чеклисты, фото, подписи
- Digital Permit to Work
- Remote Collaboration (видео, AR-подсказки)
- Автоматизированный контроль SLA

## 3) Ключевые роли

- CEO
- CTO (Digital Platform)
- VP Operations
- Chief Data & AI Officer
- Head of HSE & Compliance
- Product Owners по направлениям (drilling, production, reliability)
- Data Scientists, ML Engineers, Data Engineers
- Domain Engineers (бурение, добыча, механика, энергетика)
- Customer Success Managers

## 4) RACI (сокращенно)

- Модельный релиз: CDAO (A), MLOps (R), Domain Lead (C), CTO (C)
- Изменения на объекте: VP Ops (A), Field Supervisor (R), HSE (C)
- Коммерческий оффер: Sales Director (A), Finance (C), Product (C)

## 5) Процесс delivery

1. Диагностика актива (1–2 недели)
2. Интеграция данных и baseline-модели (2–4 недели)
3. Пилот с KPI-контрактом (8–12 недель)
4. Масштабирование и стандартизация (квартал)

## 6) Метрики операционного здоровья

- On-time interventions
- False-positive / false-negative AI alerts
- Mean Time to Decision
- Incident-free days
- Value realization rate
