# 07. Риски, HSE и governance

## 1) Реестр ключевых рисков

- Data quality risk
- OT/IT cyber risk
- Непринятие AI инженерными командами
- Регуляторные ограничения передачи данных
- Vendor lock-in

## 2) Митигации

- Data contracts + автоматические проверки качества
- Сегментация сети, SOC, регулярные pentest
- Explainable AI + обучение инженеров + dual-control
- Локализация данных и гибридные deployment-модели
- Портируемая архитектура и стандартные интерфейсы

## 3) HSE-by-design

- Digital Permit to Work обязателен для полевых задач
- AI-рекомендации не обходят safety-барьеры
- Критичные решения только через human approval
- Еженедельный разбор near-miss и corrective actions

## 4) AI governance

- Модельный комитет (Ops + AI + HSE)
- Версионирование моделей и решений
- Политика допустимых автономных действий
- Обязательная трассируемость: данные -> модель -> рекомендация -> действие -> результат
